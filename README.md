# АмурГид — калькулятор пошлины и гид по переходу Благовещенск—Хэйхэ

Статический сайт без сборки — HTML/CSS/JS. Деплоится за 5 минут.

## Структура

```
heihe-guide/
├── index.html          — главная страница
├── calculator.html     — калькулятор пошлины (3 шага)
├── guide.html          — гид по границе
├── about.html          — страница «О проекте»
├── privacy.html        — политика конфиденциальности
├── 404.html            — страница «не найдено»
├── styles.css          — все стили и дизайн-токены
├── script.js           — логика калькулятора + курс валют
├── cookie-consent.js   — баннер о cookie (работает уже сейчас, до подключения Метрики)
├── limits.json         — таможенные лимиты (редактируется без правки кода)
└── README.md
```

## Как выложить на GitHub

1. Создайте новый репозиторий на github.com (например, `heihe-guide`)
2. В терминале, находясь в папке проекта:
   ```
   git init
   git add .
   git commit -m "Первая версия сайта"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/heihe-guide.git
   git push -u origin main
   ```

## Как задеплоить (GitHub Pages)

1. Зайдите в репозиторий на github.com → **Settings → Pages**
2. Source: **Deploy from a branch** → выберите `main`, папку `/ (root)`
3. Сохраните — через 1-2 минуты сайт будет доступен по адресу вида
   `https://ВАШ_ЛОГИН.github.io/heihe-guide/`
4. `404.html` в корне репозитория GitHub Pages подхватывает автоматически —
   ничего дополнительно настраивать не нужно

Если позже понадобится хостинг без префикса `/heihe-guide/` в адресе или
собственный домен — можно перейти на Cloudflare Pages (Connect to Git,
build command и output directory оставить пустыми) без переписывания кода.

## Как обновлять данные о лимитах

Все таможенные цифры — в `limits.json`. Чтобы обновить лимит,
меняете значение в файле, коммитите и пушите — сайт обновится
автоматически (Cloudflare Pages передеплоивает при каждом push).

## Как включить рекламу

В `styles.css` и на страницах есть блоки `.ad-slot` с атрибутом
`data-ads-enabled="false"`. Чтобы включить рекламный блок — меняете
на `"true"` и вставляете контент внутрь div.

## Важно перед запуском

- `robots.txt` и `sitemap.xml` содержат placeholder-домен `amurgid.ru` —
  замените на реальный адрес сайта на GitHub Pages
  (`https://ВАШ_ЛОГИН.github.io/heihe-guide/`) сразу после первого деплоя,
  а позже — на свой домен, если купите (часть 9 плана)
- OG-теги (`og:image` и т.д.) в `<head>` каждой страницы указывают на
  `og-image.png` — файл уже есть в корне, просто проверьте, что он
  доступен по абсолютному URL после деплоя
- Баннер cookie (`cookie-consent.js`) уже работает на всех страницах.
  Когда подключите Яндекс.Метрику — просто вставьте её код счётчика в
  `<head>` каждой страницы, в баннере и политике конфиденциальности
  ничего менять не нужно
